# 提示词框架方法论

提示词工程中常用的结构化框架，选一个适合你任务的即可。

---

## CO-STAR 框架

适合：**复杂任务、需要精准风格控制**

```
Context（背景）:    你在什么情境下发出这个请求？
Objective（目标）:  你希望 AI 完成什么？
Style（风格）:      希望以什么文风/语气输出？
Tone（语调）:       正式 / 轻松 / 专业 / 幽默？
Audience（受众）:   输出面向谁？
Response（格式）:   希望以什么格式返回？
```

---

## RISEN 框架

适合：**角色扮演、专业顾问类任务**

```
Role（角色）:         让 AI 扮演什么角色？
Instructions（指令）: 具体要做什么？
Steps（步骤）:        需要按什么步骤执行？
End goal（目标）:     最终结果是什么？
Narrowing（约束）:    有哪些限制条件？
```

---

## CRISPE 框架

适合：**技术文档、分析报告类任务**

```
Capacity（能力）:    AI 应具备什么专业能力？
Role（角色）:        扮演什么身份？
Insight（背景）:     提供什么背景知识？
Statement（任务）:   具体要做什么？
Personality（风格）: 以什么个性/风格表达？
Experiment（尝试）:  要求给出多个版本/方案
```

---

## Chain-of-Thought（思维链）

适合：**逻辑推理、数学、复杂决策**

```
在提示词末尾加上：
"请一步一步思考。"
"Let's think step by step."
```

---

## Few-Shot（少样本示例）

适合：**格式固定、风格精准控制**

```
以下是示例：
输入：[示例输入1]
输出：[示例输出1]

输入：[示例输入2]
输出：[示例输出2]

现在请处理：
输入：[真实输入]
输出：
```

---

## System + User 分层框架

适合：**API 调用、Agent 构建**

```
[System Prompt]
设定角色、规则、限制、返回格式

[User Prompt]
具体的单次任务指令
```
